遵循[常规升级流程](./README.md)。

## 升级后显著变化

### API 变化

- 向 `PropagationPolicy`（传播策略）和 `ClusterPropagationPolicy`（集群传播策略）添加 `SchedulePriority`（调度优先级）字段来设置应用的优先级，以控制应用调度的顺序：
    - 支持多种优先级类来源（Kubernetes、Pod模板、联邦 [后续版本考虑是否实现]）
    - 可配置的优先级类解析策略
    - 保持与Kubernetes优先级行为的兼容性
- 向 `ResourceBinding`（资源绑定）API添加 `SchedulePriority`（调度优先级）、`Priority`（优先级）和 `PreemptionPolicy`（抢占策略）字段，以支持基于优先级的调度和抢占控制。
    - 优先级：定义调度优先级（值越高 = 优先级越高）
    - 抢占策略：控制绑定是否可以抢占较低优先级的绑定
- `API变更`：在 `Karmada` API中引入了 `PriorityClassName`（优先级类名称），将用于指定该组件的优先级类名称。
- `API变更`：在 `ResourceBinding`（资源绑定）和 `ClusterResourceBinding`（集群资源绑定）中都引入了 `Scheduling`（调度）暂停功能，供第三方系统用于暂停应用程序调度。
- `karmada-operator`：作为Karmada API服务器组件配置的一部分，新的 `sidecarContainers`（边车容器）字段可用于选择性地为Karmada API服务器指定边车容器。

### 弃用

- 由于 `grpc.DialContext` 和 `grpc.WithBlock` 已被弃用，需使用 `grpc.NewClient` 来替代它们，同时保持原有功能。
- `karmadactl`：`register` 命令的 `--ca-cert-path` 标志已被标记为弃用，因为它从未被使用过，并且会在未来的版本中移除。

### karmada-controller-manager

- Karmada Lua 解释器将不再支持 Lua 函数 `string.rep` 和 `string.gsub`。一般来说，这些函数在自定义的 Karmada 资源解释器中并不常用。由于存在潜在的安全风险，它们已被禁用。在升级之前，请检查你的 Lua 脚本，确认是否正在使用这些函数。如果使用了，请用其他实现方式来替代它们。
